PEAXAI_counterfactuals(data, x, y, final_model, efficiency_thresholds, directional_vector, n_expand, n_grid, max_y, min_x) provides, for each DMU in data, the minimum counterfactual input reductions and output expansions required to be classified as efficient at the selected probability thresholds given a direction. The function returns the corresponding counterfactual targets, the associated inefficiency estimates, and the probability attained under the selected directional vector. This design is consistent with the probabilistic efficiency framework, where inefficiency is defined as the minimum modification needed for a DMU to reach a given efficiency-probability threshold.
In PEAXAI_counterfactuals(), the argument directional_vector is a list that defines the direction used in the counterfactual search. In particular, its component relative_importance contains the variable weights, which can be obtained data-driven using PEAXAI_global_importance() or PEAXAI_local_importance(), or alternatively supplied by the user. The component baseline defines the reference point used to construct the directional vector, such as the sample mean ‘mean’, a vector of ones ‘ones’, or the observed values of each DMU ‘self’. Regarding the search procedure, n_expand determines how the initial search interval is expanded, n_grid specifies how many candidate counterfactuals are evaluated at each iteration, and max_y and min_x set the maximum output expansion and minimum input contraction limits to ensure the generated targets remain realistic.
